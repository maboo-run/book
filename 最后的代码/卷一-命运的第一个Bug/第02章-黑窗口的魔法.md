第二天，林北早早地来到了教室。

他昨晚激动得半夜都没睡着，脑子里一直在想着那个神奇的"Hello World"。就那么简单的一行代码，竟然能让电脑听从自己的指令！这种感觉，就像是第一次学会骑自行车，虽然只是简单的前进，但那种掌控感让人上瘾。

教室里已经有几个同学在了。李明依然坐在老位置，面前摆着厚厚的书和笔记本，正在奋笔疾书。王胖子也来了，正在啃包子，油渍滴在键盘上也浑然不觉。

苏晴还是坐在靠窗的位置，安静得像一幅画。

"早啊，林北！"王胖子嘴里塞着包子，含糊不清地打招呼，"昨晚想了没？是不是也跟我一样，觉得编程也太简单了？"

林北笑了笑："还行吧，不过感觉后面应该会难起来……"

"怕什么！"王胖子大大咧咧地说，"我昨晚回去试了好几遍Hello World，一次都没出错！我觉得我有天赋！"

李明抬起头，推了推眼镜："打印语句确实很简单，但那只是最基础的。真正的编程，要考虑算法效率、内存管理、并发控制……"

"打住打住！"王胖子摆摆手，"你说的这些我一个字都听不懂。反正今天马老师会教新东西，到时候再说呗。"

就在这时，马老师走进了教室，手里拿着一个保温杯，精神看起来不错。

"同学们，今天我们要学习一个新技能。"马老师打开了投影，"昨天我们用的是Python的交互式界面，今天我们要学习真正程序员的工具——命令行。"

"命令行？"有同学小声问。

"对，也叫终端、控制台、Terminal……名字有很多，但本质都是一样的。"马老师切换到了一个全黑的窗口。

屏幕上瞬间漆黑一片，只有一个孤零零的白色光标在闪烁，像是在黑夜里眨着的眼睛。

没有图标，没有鼠标指针，没有绚丽的背景图。只有黑底白字，冷峻得像是某种古老的仪式。

林北盯着那个黑窗口，心里突然有种莫名的紧张感。这就是电影里黑客用的那种吗？

"这……这也太酷了吧！"王胖子两眼放光，"就像《黑客帝国》里那种！"

"差不多。"马老师神情严肃，声音里透着一丝敬畏，"在这个窗口里，你不再是那个只会点鼠标的用户。你是指挥官，是上帝。"

"在这里，你可以直接和电脑的灵魂对话。浏览文件、运行程序、安装软件、甚至……"马老师顿了顿，"甚至可以毁灭系统。"

教室里突然安静了下来。

毁灭系统？林北感觉手心出汗了。这听起来既刺激又危险。

"当然，"马老师笑了笑，"前提是你知道对应的命令。命令行就像是一把双刃剑，用得好，事半功倍；用不好……轻则文件丢失，重则系统崩溃。"

"那我们会不会把电脑搞坏啊？"前排一个女生怯怯地问。

"有可能。"马老师诚实地说，"所以今天我要先教你们一些基础命令，让你们熟悉这个环境。记住，在命令行里，没有撤销键，没有后悔药。一旦按下回车，命令就执行了。"

林北紧张地咽了口唾沫。

"首先，我们来学习几个最基础的命令。"马老师开始在黑窗口里敲击。

"第一个命令，`pwd`。Print Working Directory，打印当前工作目录。也就是告诉你，你现在在哪儿。"

```bash
pwd
```

屏幕上跳出一行路径：

```
/Users/zhangma
```

"看，这就是我当前所在的位置。就像你在一个巨大的迷宫里，`pwd`就是你的GPS，告诉你坐标。"

"第二个命令，`ls`。List，列出当前目录下的文件和文件夹。"

```bash
ls
```

屏幕上瞬间出现了一堆文件名：

```
Desktop    Documents    Downloads    Pictures    Music
```

"这就像是你打开一个房间，环顾四周，看看房间里有什么东西。"

林北跟着操作，在自己的电脑上敲下`ls`，屏幕上也显示出了类似的文件列表。

"第三个命令，`cd`。Change Directory，切换目录。这是最常用的命令之一。"

```bash
cd Desktop
```

"现在我就从家目录进入了桌面目录。就像你从客厅走到了卧室。"

林北照做了，输入`cd Desktop`，然后再输入`pwd`，果然路径变了：

```
/Users/linbei/Desktop
```

"太神奇了！"林北小声嘀咕。虽然只是简单的命令，但这种直接操控电脑的感觉，比点鼠标爽多了。

"第四个命令，`mkdir`。Make Directory，创建目录。"

```bash
mkdir my_first_project
```

"这就在桌面上创建了一个新文件夹。"

林北敲下这行命令，屏住呼吸等待着。

屏幕上没有任何反应。光标依旧冷漠地闪烁着，仿佛什么都没发生。

"坏了？没成功？"林北心里一沉。

马老师似乎听到了他的嘀咕，笑着说："同学们，记住一句话——在命令行里，没有消息就是最好的消息。"

"它只在你出错的时候说话。没报错，就说明成功了。"

林北赶紧切回桌面看了一眼，果然多了一个叫`my_first_project`的文件夹。

"……这脾气也太大了吧？"林北吐槽道。成功了也不吭一声，太高冷了。

但转念一想，这种高冷的风格，确实有种莫名的帅气感。

"好了，现在大家都跟着练习一遍。"马老师说，"创建一个文件夹，进入那个文件夹，然后用`pwd`确认你的位置。"

教室里响起了噼里啪啦的键盘声。

林北按部就班地操作：

```bash
mkdir my_first_project
cd my_first_project
pwd
```

一切顺利。

但就在这时，王胖子那边传来了一声惨叫："卧槽！报错了！"

众人纷纷看向他。

只见王胖子的屏幕上显示着一行红色的错误信息：

```
mkdir: cannot create directory 'my first project': No such file or directory
```

"怎么回事？"王胖子抓耳挠腮，"我明明照着老师的输入了啊！"

马老师走过去看了一眼，笑了："你的文件夹名字里有空格。"

"空格怎么了？不行吗？"王胖子不解。

"在命令行里，空格是分隔符。"马老师解释道，"你输入`mkdir my first project`，电脑会以为你要创建三个文件夹：`my`、`first`、`project`。"

"那怎么办？我就想要一个带空格的名字！"王胖子不服。

"有两个办法。第一，用引号把名字括起来：`mkdir "my first project"`。第二，用下划线或短横线代替空格：`mkdir my_first_project`。"

王胖子恍然大悟，重新输入了一遍，这次成功了。

"原来还有这种坑……"林北若有所思。看来命令行虽然强大，但也处处是陷阱。

"接下来，我们来学习一个更重要的命令——`python`。"马老师说，"这个命令可以运行Python程序。"

"昨天我们用交互式界面，每敲一行就执行一行。但实际工作中，我们要把代码写在文件里，然后一次性运行整个文件。"

马老师打开了一个文本编辑器（VS Code），那种深色的背景和彩色的代码高亮，瞬间让林北觉得专业度拉满。

"我们在刚才创建的文件夹里，写一个真正的Python程序。"

马老师新建了一个文件，命名为`hello.py`，然后输入：

```python
name = input("请输入你的名字：")
print("你好，" + name)
```

"保存。记住，`.py`后缀很重要，它告诉电脑这是一个Python程序，不是普通的文本文件。"

林北照做，创建了同样的文件。

"现在，回到命令行，用`python`命令运行这个文件。"

```bash
python hello.py
```

回车。

屏幕上显示：

```
请输入你的名字：
```

光标停住了，在闪烁，等待着。

林北颤抖着手，输入了"林北"，然后按下回车。

```
你好，林北
```

"成功了！"林北激动得差点跳起来。

这和昨天那种自言自语完全不同。程序停下来等他，接收了他的名字，然后把他的名字嵌进了句子里还给他。

这就是交互！这就是对话！

林北感觉自己仿佛打通了任督二脉。

"老师，我有个问题！"李明举手，"这个`input`函数返回的是什么类型？"

马老师点头："好问题。`input`返回的永远是字符串类型。"

"那如果我想让用户输入数字呢？"李明追问。

"需要类型转换。用`int()`或`float()`函数。"

李明若有所思地点点头，立刻开始尝试。

林北则在想另一个问题。既然能接收名字，那能不能做点更厉害的事？

他想起昨天遇到的那个问题——让程序帮他做数学题。

"老师，我想做一个加法器！"林北兴奋地说。

"很好的想法，试试看。"马老师鼓励道。

林北信心满满地修改代码：

```python
num = input("请输入一个数字：")
print(num + 1)
```

他觉得这逻辑完美。用户输入一个数字，程序自动加1，多简单！

保存，运行，输入"5"。

然而，屏幕上没有出现期待的"6"，而是出现了一大堆红色的英文，像血一样刺眼：

```
TypeError: can only concatenate str (not "int") to str
```

林北傻眼了："这是什么意思？又出错了？"

他仔细检查代码，`num`没拼错，`+`号也对，为什么会报错？

"因为类型不匹配。"苏晴的声音从旁边传来。

林北转头，发现苏晴不知什么时候走了过来。她指着屏幕上的错误信息："TypeError，类型错误。你输入的'5'是字符串，不是数字。"

"可是……5不就是5吗？"林北一脸懵逼。

"在电脑眼里，'5'和5是两种完全不同的东西。"苏晴解释道，声音很温柔，"一个是字符，一个是数字。就像这个数字5，"她在纸上写了个5，"和5块钱，"她拿出一个硬币，"能一样吗？"

林北恍然大悟："所以我要先把字符串'5'转换成数字5？"

"对。用`int()`函数。"苏晴点点头。

林北立刻修改代码：

```python
num = int(input("请输入一个数字："))
print(num + 1)
```

再次运行，输入"5"。

```
6
```

"成功了！谢谢你，苏晴！"林北激动地说。

苏晴微微一笑，回到了自己的座位。

林北看着屏幕上那个"6"，心里美滋滋的。虽然只是一个简单的加法，但这是他第一次让程序"思考"，让程序进行计算。

就在林北沉浸在成功的喜悦中时，王胖子那边又传来了动静。

"老师！老师！我的电脑卡住了！"王胖子的声音带着哭腔。

众人看过去，只见王胖子的屏幕上什么都没有，只有一个闪烁的光标，而且鼠标点哪儿都没反应。

"你干了什么？"马老师走过去问。

"我……我也不知道啊！"王胖子慌了，"我就想试试那个`cd`命令，然后就卡住了！"

马老师看了看他的屏幕，又看了看他刚才敲的命令，忍不住笑了。

"你输入了什么？"

"就是`cd`啊，然后按了回车……哦，好像还按了几次……"

马老师摇摇头："你单独输入`cd`然后回车，程序在等你输入目标路径。但你又按了几次回车，所以现在它卡在那里等着你的输入。"

"那怎么办？"王胖子快哭了。

"按Ctrl+C，强制中断。"

王胖子照做，按下Ctrl+C，屏幕瞬间恢复正常，光标重新出现。

"呼……"王胖子长舒了一口气，"吓死我了，我还以为电脑坏了。"

"Ctrl+C是命令行里的急刹车。"马老师说，"如果程序卡住了，或者你想中断执行，就按这个组合键。记住了吗？"

"记住了！"王胖子拍着胸口，"以后我再也不乱按了。"

但他这话才说完不到五分钟，又传来了惨叫。

"我靠！这次是真的坏了！"

众人再次看过去，只见王胖子的屏幕上密密麻麻全是文件名，刷得飞快，像瀑布一样停不下来。

```
file1.txt
file2.txt
file3.txt
file4.txt
...
```

"我就输了个`ls -R`，然后它就疯了！"王胖子手足无措。

马老师这次没笑，而是快步走过去，按下了Ctrl+C。

屏幕停止了刷动。

"你输入`ls -R`，是递归列出所有子目录的文件。"马老师解释道，"如果你在系统根目录下执行这个命令，它会列出整个硬盘的所有文件。几万个文件，当然停不下来。"

"我……我不知道啊……"王胖子欲哭无泪。

"所以说，命令行不能乱试。"马老师严肃地说，"每个命令都有它的作用域和影响范围。在不了解的情况下，千万不要随便加参数，更不要在系统目录下执行危险命令。"

林北在心里默默记下：命令行虽然强大,但也很危险。一定要知道自己在干什么。

"不过，王胖子同学也不用太紧张。"马老师话锋一转，"犯错是学习的一部分。只要不是执行`rm -rf /`这种自杀命令,一般都不会有大问题。"

"`rm -rf /`是什么？"有同学好奇地问。

"删除根目录下的所有文件,包括系统文件。"马老师淡淡地说，"执行这个命令，你的系统就彻底废了。"

教室里倒吸一口凉气。

"所以，大家记住，永远不要执行不理解的命令。网上有些人喜欢恶作剧，告诉新手'这个命令可以加速系统'，实际上是让你删除重要文件。"

林北咽了咽口水。原来编程的世界里，处处是坑。

"不过也不用太怕。"马老师笑了笑，"只要你遵循一个原则——在执行命令前，先想想它会产生什么后果——就不会出大问题。"

接下来的时间，马老师让大家自由练习。

林北尝试了各种命令组合，创建文件夹、进入文件夹、查看文件、运行程序……渐渐地，他开始享受这种在黑窗口里指挥电脑的感觉。

虽然偶尔会输错命令，看到报错信息。但每次解决一个报错，他都会有一种成就感。

李明则在尝试更高级的操作，比如用管道符连接命令、用通配符批量处理文件……虽然经常搞得一团糟，但他乐在其中。

王胖子吃一堑长一智，开始小心翼翼地操作，每输入一个命令都要先问问旁边的同学。虽然笨拙，但至少没再把系统搞崩。

苏晴依然安静地坐在角落，但林北注意到，她的屏幕上已经运行了好几个复杂的程序,代码写得又快又准。

课程快结束的时候，马老师布置了一个小任务。

"大家回去后，用命令行创建一个自己的项目文件夹，然后在里面写一个程序，实现简单的用户交互。可以是问候、计算、或者其他任何你想到的功能。"

"下节课，我们会学习一个新的概念——变量。它是编程的基础，也是理解程序运行的关键。"

林北收拾东西的时候，脑子里已经在构思自己的小程序了。

一个简单的自我介绍程序？还是一个年龄计算器？或者……做一个猜数字游戏？

"林北，一起走？"王胖子拍了拍他的肩膀。

"好啊。"林北背起包。

两人走出教室，王胖子一边走一边感叹："今天可真是惊心动魄啊！差点把电脑搞坏。"

"你啊，就是太莽了。"林北笑道，"下次输命令前先想想会发生什么。"

"我知道了。"王胖子挠挠头，"不过说实话,那个黑窗口还挺好玩的。虽然经常报错，但一旦成功了，那种感觉……"

"特别爽，对吧？"林北接话。

"对对对！"王胖子兴奋地说，"就像打游戏通关一样！"

两人走到地铁站，分别的时候，王胖子突然问："你说，咱们真的能学会编程吗？三个月后真的能找到工作吗？"

林北愣了一下,然后坚定地说："能。我觉得咱们选对了。虽然现在还有很多不懂的，但至少……我们已经开始了。"

"说得对！"王胖子竖起大拇指，"那咱们一起加油！"

"一起加油！"

回到出租屋，林北打开电脑,开始写今天的作业。

他在桌面上创建了一个文件夹，命名为"LinBei_Projects"，然后进入文件夹，新建了一个`intro.py`文件。

他想做一个自我介绍程序，可以问用户的名字、年龄、爱好，然后给出回应。

```python
name = input("你叫什么名字？")
age = input("你多大了？")
hobby = input("你的爱好是什么？")

print("原来你叫" + name + "啊！")
print("才" + age + "岁，真年轻！")
print("我也喜欢" + hobby + "！")
```

保存，运行。

```
你叫什么名字？林北
你多大了？22
你的爱好是什么？打游戏
原来你叫林北啊！
才22岁，真年轻！
我也喜欢打游戏！
```

林北看着屏幕上的输出，忍不住笑了。

虽然这个程序很简陋，虽然它只是简单地重复用户的输入，但这是他独立完成的第一个交互程序。

从昨天的"Hello World"，到今天的用户交互，他感觉自己已经迈出了一大步。

但他也意识到，自己还有太多不懂的东西。比如，怎么让程序判断用户输入的是不是数字？怎么让程序根据不同的输入做出不同的反应？

这些问题，让他既困惑又兴奋。

他想起马老师说的，下节课要学"变量"。

变量是什么？听起来很神秘。

林北打开浏览器，搜索了一下"Python变量"。

页面上出现了大量的解释："变量是存储数据的容器"、"变量是内存中的一个位置"、"变量可以保存不同类型的值"……

林北越看越晕。
