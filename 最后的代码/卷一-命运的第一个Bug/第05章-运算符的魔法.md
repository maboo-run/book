"计算机用二进制存储浮点数。就像1/3在十进制里是0.333...,无法精确表示一样,0.1在二进制里也是无限循环小数。"

"好问题!"马老师赞许地点头,"所以在金融计算中,我们通常有两种做法:"

"第一,把金额转换成整数（分）来计算。比如10.5元存成1050分。"

"第二,使用专门的`Decimal`模块,它能提供更高精度的十进制运算。"

```python
from decimal import Decimal

a = Decimal("0.1")
b = Decimal("0.2")
print(a + b)  # 输出: 0.3
```

"所以,记住:永远不要用`==`直接比较两个浮点数是否相等。"

```python
# 错误的做法
if 0.1 + 0.2 == 0.3:
    print("相等")  # 不会执行

# 正确的做法
if abs((0.1 + 0.2) - 0.3) < 0.00001:
    print("足够接近")  # 会执行
```

"记住,括号的优先级最高,然后是幂运算,然后是乘除,最后是加减。"

```python
age = 18
```

"比较运算符的结果永远是布尔值:True或False。"

"注意,判断相等用双等号`==`,不是单等号`=`。"

"这是新手最容易犯的错误!"马老师强调道,"一个等号是赋值,两个等号是比较。"

```python
# 赋值

# 比较
a == 10  # 判断a是否等于10,返回True或False
```

"有些情况会报错,有些情况会默默地改掉你的数据。"马老师说,"比如:"

```python
a = 10
if a = 10:  # 错误!这是赋值,不是比较
    print("相等")
```

"这会报错:`SyntaxError: invalid syntax`。"

"但在有些语言里,这种写法不会报错,而是会把10赋给a,然后判断True或False。非常危险。"

"所以,记住:比较用双等号,赋值用单等号。永远不要搞混。"

林北在笔记本上用红笔写下:比较==,赋值=,千万别搞混!

"最后,我们学习逻辑运算符。"马老师说。

```python
# and:两个都为True,结果才为True

# not:取反
print(not True)   # False
print(not False)  # True
```

"逻辑运算符用于组合多个条件。"

"比如,你想判断一个用户是否可以进入网吧:年龄大于等于18岁,并且有身份证。"


"或者,判断一个人是否有优惠:是学生或者是老人。"

```python

```

苏晴举手:"老师,`and`和`or`可以连用吗？"

"当然可以。但要注意优先级。"马老师说。

"`and`的优先级高于`or`。"

```python

```

"如果你想先算`or`,就要用括号:"

```python
result = (True or False) and False

```

"所以,复杂的逻辑表达式,最好也用括号明确优先级。"
