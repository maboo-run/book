李明立刻举手，一副胸有成竹的样子："变量就是在内存中开辟的一块存储空间，用于存放运算过程中需要用到的数据。"

林北听得一愣一愣的。内存？存储空间？这都是什么和什么？

马老师笑了笑："李明同学的回答很专业，但是……"他顿了顿，"有多少同学听懂了？"

教室里一片沉默。

"这就是理论和实践的差距。"马老师说，"书上的定义是对的，但对新手来说，太抽象了。"

他转身，在白板上画了一个简单的盒子。

"我给你们举个例子。假设你在玩数学题,要算`5 + 3 = 8`,然后再算`8 × 2 = 16`。"

"中间那个8,你怎么记住的？"

"用草稿纸！"王胖子脱口而出。

"对！"马老师拍手,"变量就是电脑的草稿纸！"

"你做数学题的时候,会把中间结果写在草稿纸上，对吧？比如你在纸上写'8',然后下一步用这个8去乘2。"

"变量就是这样。我们给这块'草稿纸'起个名字,比如叫`result`,然后把8写在上面。等需要的时候,再从`result`这张纸上取出8来用。"

林北恍然大悟。原来这么简单！草稿纸,这个比喻太贴切了。

"让我来演示一下。"马老师打开了编辑器。

```python
# 第一步：定义变量,相当于拿出一张草稿纸,在上面写个数字
age = 22

# 第二步：使用变量
print(age)  # 输出：22

# 第三步：修改变量,相当于擦掉原来的数字,写个新的
age = 23
print(age)  # 输出：23
```

"看到了吗？变量就像一个盒子，或者说一张可以反复擦写的草稿纸。你可以往里面放东西,也可以把东西拿出来用,还可以换掉里面的东西。"

林北若有所思地点点头。这个比喻比李明说的"内存空间"好理解多了。

"老师，我有个问题！"李明又举手了,"为什么叫`age`就能代表年龄？电脑怎么知道这是年龄？"

"好问题。"马老师说,"其实电脑根本不知道这是年龄。`age`只是一个标签,一个名字。你叫它`age`，叫它`nianling`，甚至叫它`xiaoming`,电脑都不在乎。"

"那为什么要叫`age`?"

"为了让人看懂。"马老师强调道,"代码不仅是写给电脑看的,更是写给人看的。好的变量名能让代码一目了然，烂的变量名会让代码变成天书。"

马老师举了两个例子：

```python
# 烂代码
a = 22
b = "林北"
c = True

# 好代码
age = 22
name = "林北"
is_student = True
```

"看出区别了吗？第一段代码,你完全不知道a、b、c是干什么的。第二段代码,一眼就能看出age是年龄，name是姓名，s_student是是否为学生。"

"所以，"马老师郑重地说,"给变量起个好名字，是程序员的基本素养。"

"那变量命名有规则吗？"苏晴突然开口问，这是她第一次主动提问。

所有人都看向她,林北注意到她的表情很认真,眼神里带着求知欲。

"当然有！"马老师立刻来了精神,"Python的变量命名规则有几条铁律："

他在白板上写下：

1. 只能包含字母、数字、下划线
2. 不能以数字开头
3. 不能使用关键字（比如if、else、for等）
4. 区分大小写（Age和age是两个不同的变量）

"另外，虽然不是强制要求，但有一些约定俗成的规范："

- 多个单词用下划线连接：`user_name`、`total_score`
- 常量全大写：`MAX_SIZE`、`PI`
- 布尔类型用is开头：`is_student`、`has_permission`
- 避免使用单个字母（除了循环变量i、j、k）

"记住这些规则了吗？"马老师问。

"记住了！"大家齐声回答。

"好，那我来考考你们。"马老师在白板上写了几个变量名：

```python
user_name    # 对吗？
2nd_place    # 对吗？
for          # 对吗？
userName     # 对吗？
user-name    # 对吗？
_private     # 对吗？
```

林北盯着这些名字，开始思考。

`user_name`应该是对的，符合规则……

`2nd_place`以数字开头，应该不行……

`for`是关键字，肯定不行……

`userName`用的是驼峰命名，虽然在Python里不太常见,但应该也可以吧？

`user-name`有短横线，短横线好像不在允许的字符里……

`_private`以下划线开头，这个……不确定。

"李明，你来说说。"马老师点名。

李明站起来,推了推眼镜，一副自信满满的样子:"让我来给大家科普一下。"

"第一个`user_name`，完全符合PEP 8规范，正确。"

"第二个`2nd_place`，违反了'不能以数字开头'的规则，错误。应该写成`second_place`或`place_2nd`。"

"第三个`for`，这是Python的关键字，保留字，不能用作变量名，错误。"

"第四个`userName`，采用驼峰命名法，在Python里不推荐，但语法上是合法的。在Java或JavaScript里这样写才是标准的。"

"第五个`user-name`，包含短横线，短横线在Python里是减号运算符，所以这会被解析成`user`减`name`，错误。"

"第六个`_private`，以下划线开头……"李明顿了顿，"这个涉及到Python的命名约定。单下划线开头表示这是内部使用的变量，虽然语法上合法，但有特殊含义……"

他越说越快，越说越兴奋："如果是双下划线开头，比如`__private`，那就触发了名称改编机制，会被解释器改成`_ClassName__private`的形式，这是为了实现伪私有属性……还有前后都带双下划线的，比如`__init__`，那是魔术方法，也叫dunder方法……"

林北听得头晕眼花。什么改编机制？什么魔术方法？这都是啥啊？

王胖子更是直接趴在了桌上："我的天,这也太复杂了吧？我就想起个变量名而已……"

"停停停！"马老师打断了李明,"李明同学,你讲得太深了。"

"重要，但不是现在。"马老师说,"你记住一句话：学习是循序渐进的。现在大家连变量是什么都还没完全理解，你就跟他们讲名称改编？"

李明这才反应过来，有些不好意思地坐下。

"还记得我说的吗？变量是可以改变的。"

马老师写下一段代码：

```python
score = 85

score = score + 10
print("加分后：", score)

score = score + 5
print("再加分后：", score)
```

输出：

```

加分后： 95
再加分后： 100
```

"看，`score`的值在不断变化。这就是变量的精髓——它是'可变'的。"

李明举手："老师，`score = score + 10`这个语句，从数学角度看是错误的。等号左边的score和右边的score怎么能相等？"

"好问题！"马老师赞许地点头,"这就是编程和数学的区别。"

"在数学里，等号表示'相等'。但在编程里，等号表示'赋值'。"

"也就是说，`score = score + 10`的意思是：先算出`score + 10`的结果，然后把这个结果重新赋给`score`。"

马老师在白板上画了个流程图：

```
1. 取出score的值：85
2. 计算 85 + 10 = 95
3. 把95放回score这个盒子里
4. 原来的85被覆盖，现在score是95
```

"这就像你在草稿纸上，先看了看写的是85，心算出85+10=95，然后擦掉85，写上95。"

林北恍然大悟。原来编程里的等号和数学里的等号意思不一样！

"为了避免混淆，有些编程语言用`:=`或`<-`来表示赋值。"马老师补充道，"但Python沿用了等号，所以大家要习惯这个逻辑。"

"老师，我有个问题。"苏晴再次开口，"如果我想交换两个变量的值，应该怎么做？"

"比如，有`a = 5`和`b = 10`，我想让`a`变成10，`b`变成5。"

"好问题！"马老师眼睛一亮，"这是个经典问题。大家想想，怎么做？"

林北思考了一下。如果直接写：

```python

b = a
```

那`a`会变成10，但`b`也会变成10，因为`a`已经被改成10了……

"需要一个中间变量！"李明抢答，"就像交换两杯水，需要一个空杯子。"

"完全正确！"马老师写下：

```python

# 需要一个临时变量

b = temp  # b = 5
```

"这是最基础的方法。不过，Python有个更简洁的写法："

```python
a, b = b, a
```

"一行代码就搞定了。这是Python的元组解包特性，以后我们会详细讲。"
